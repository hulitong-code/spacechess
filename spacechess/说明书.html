🐍 Python 文件说明 Python Files

main.py  — 主程序入口— Entry point

位于项目根目录，直接运行启动游戏（python main.py）。 它将 src/ 加入 sys.path，然后导入其他模块。 负责 pygame 主循环、所有鼠标/键盘事件的分发、按钮绘制与交互、以及游戏状态的驱动。

Lives at the project root; run it to start the game (python main.py). It adds src/ to sys.path, then imports the other modules. Owns the pygame main loop, all mouse/keyboard event dispatch, button rendering and interaction, and game-state driving.

职责范围：事件循环 → 路由到正确的 GamePhase 处理器 → 调用 ui.py 的 draw_* 函数 → flip 屏幕。游戏状态本身由 game.py 维护。 Scope: event loop → route to the correct GamePhase handler → call ui.py draw_* functions → flip screen. Game state is owned by game.py.
src/data.py 数据data
所有静态游戏数据的唯一来源： Single source of truth for all static game data:
  • BOARD — 32 格棋盘定义（名称、英文名、格子类型）
  • BOARD — 32-cell board definition (name, English name, cell type)
  • PLANETS — 19 颗星球（从 data/星球信息.md 启动时读取）
  • PLANETS — 19 planets (parsed from data/星球信息.md at startup)
  • EVENT_CARDS / CHANCE_CARDS — 各 14 张卡牌
  • EVENT_CARDS / CHANCE_CARDS — 14 cards each
  • COLORS — 六位玩家颜色、UI 调色板
  • COLORS — 6-player colors, UI palette
  • MAX_PLAYERS = 6、STARTING_FUNDS、PASS_EARTH_BONUS 等全局常量
  • MAX_PLAYERS = 6, STARTING_FUNDS, PASS_EARTH_BONUS and other global constants
src/game.py 逻辑logic
纯游戏逻辑，不含任何 pygame 绘制代码。包含： Pure game logic, no pygame rendering. Contains:
  • Player — 玩家属性（位置、资金、星球、状态）
  • Player — player attributes (position, funds, planets, status)
  • GamePhase — 枚举：游戏所处阶段（掷骰/购买/支付等）
  • GamePhase — enum of game phases (rolling, buying, paying, etc.)
  • GameState — 主状态机，所有规则执行逻辑都在此
  • GameState — main state machine; all rule execution lives here
  • get_player_planet_list() — 生成面板用的星球信息字典
  • get_player_planet_list() — builds the panel planet-info dicts
src/ui.py 渲染rendering
所有 pygame 绘制函数的集合，每个都以 draw_ 开头： All pygame drawing functions, each prefixed draw_:
  • init_assets() — 加载图片、预计算棋子圆头、压暗图标
  • init_assets() — load images, precompute circular token heads, darken icons
  • draw_board() — 棋盘 + 32 格棋子
  • draw_board() — board + 32 player tokens
  • draw_panel() — 右侧玩家状态面板
  • draw_panel() — right-side player status panel
  • draw_start_screen() — 选人界面（3×2 图标网格）
  • draw_start_screen() — player selection screen (3×2 icon grid)
  • draw_fund_notice() — 资金变化弹窗（双行大字）
  • draw_fund_notice() — fund change popup (two-line large text)
  • 以及卡牌弹窗、星球选择器、实验选择器等

src/lang.py 国际化i18n
中英文双语支持模块。所有界面文本均在此定义，通过以下接口访问： Bilingual (Chinese/English) support module. All UI text is defined here, accessed via:
  • t("key", **kwargs) — 按 key 返回当前语言文本— return text for current language by key
  • loc_name(d) — 从含 name/name_en 的字典返回本地化名称— localised name from a dict with name/name_en

游戏界面右下角的按钮可随时切换语言，棋盘图和玩家名称同步更新。 The in-game bottom-right button switches language at any time; board image and player names update immediately.
src/calibrate.py 工具tool
交互式棋盘格子坐标标定工具。在棋盘图上逐一点击每格的左上角和右下角，程序自动计算格子中心坐标，保存到 assets/background/calibration.json。 仅在更换棋盘图片时需要重新运行。 Interactive board-cell coordinate calibration tool. Click the top-left and bottom-right corners of each cell on the board image; the tool computes cell centres and saves them to assets/background/calibration.json. Only needs to be re-run when the board image changes.
src/calibrate_tokens.py 工具tool
角色头像圆形裁剪标定工具。对每张玩家图片，点击头部圆心和边缘，确定用于游戏棋子的圆形裁剪区域，保存到 assets/token_crops.json。 更换玩家头像时重新运行。 Player avatar circular-crop calibration tool. For each player image, click the head centre and the edge to define the circle crop used for the game token; saved to assets/token_crops.json. Re-run when player avatars are replaced.
src/crop_tool.py 工具tool
星球图片圆形裁剪工具。拖拽鼠标在 planet_1.jpg 上画圆，确认后批量将全部 19 张星球图裁剪为圆形，保存到 assets/planets/circles/。 Planet image circular-crop tool. Drag on planet_1.jpg to draw a circle; confirm to batch-crop all 19 planet images into circles, saved to assets/planets/circles/.

游戏流程

Turn Flow

  1. 轮到该玩家时，点击「掷骰子」，自动投出两颗骰子。
  2. When it's your turn, click "Roll Dice" — two dice are thrown automatically.
  3. 若投出双数（两颗相同），本轮结束后可再投一次；若连续三次双数，立即被送入空间站，停走一轮。
  4. If you roll doubles, you may roll again after resolving this turn; three consecutive doubles sends you to the Space Station immediately.
  5. 棋子按骰子总点数移动，经过地球获得 20 万奖励，精确踏上地球则可额外免费升级一颗自己的星球。
  6. Move your token by the dice total. Passing Earth grants +20; landing exactly on Earth also gives a free planet upgrade.
  7. 落在无人星球：可选择花费探索费购买；落在他人星球：必须支付过路费，之后可选择收购该星球。
  8. Land on an unclaimed planet: optionally pay the explore cost to claim it. Land on another player's planet: must pay toll, then optionally trade for it.
  9. 落在特殊格子时执行对应效果（见下方格子说明）。
  10. Landing on a special cell triggers its effect (see cell table below).
  11. 当前玩家所有行动完成后，自动切换到下一位玩家。
  12. After all actions are resolved, the turn passes to the next player.

Opponents landing on your planet must pay the toll, which increases with level. Each experiment doubles the current toll.

实验（太空实验室触发）

Experiments (triggered at Space Lab)

落在太空实验室时，可对自己持有的一颗支持实验的星球做一次实验。效果：该星球免费升一级，并且过路费翻倍。每种实验每颗星球只能做一次。

Landing on the Space Lab lets you conduct one experiment on one of your planets that supports it. Effect: free +1 upgrade level and toll ×2. Each experiment type can only be done once per planet.

收购（交易）

Trading (Acquiring a Planet)

  1. 落在他人星球，支付过路费后，可申请以收购费购买该星球的探索权。
  2. After paying the toll on an opponent's planet, you may offer the trade price to acquire it.
  3. 星球主人可以拒绝交易。
  4. The planet owner may refuse the trade.
  5. 已达 Lv 4（成功探索）的星球不可被收购。
  6. Planets at Lv 4 (Full Exploration) cannot be traded.
  7. 收购费 = 星球卡上各已完成探索等级对应收购费之和。
  8. Trade cost = sum of the trade costs for all levels already explored (from the planet card).

交公（抵押给银行）

Mortgaging (Selling to the Bank)

  1. 无力支付过路费或其他费用时，可选择将自己的星球交公给银行换取经费。
  2. When unable to pay a toll or other cost, you may mortgage your planet to the bank for funds.
  3. 交公后，该星球的所有探索项目归零，重新成为无人星球。
  4. After mortgaging, all exploration levels are reset and the planet becomes unclaimed again.
  5. 交公价 = 星球卡上各已完成探索等级对应交公价之和。
  6. Mortgage value = sum of mortgage values for all completed exploration levels.

🏆 游戏结束与破产 Game End & Bankruptcy

结束条件

End Conditions

触发条件Trigger 结果Result
所有其他玩家破产，仅剩 1 人 All others bankrupt, 1 player left 该玩家直接获胜 That player wins immediately
某一玩家破产（其余人仍在） One player goes bankrupt (others remain) 游戏结束，所有剩余玩家清算总资产，最多者获胜 Game ends; all remaining players tally net worth, highest wins

总资产计算

Net Worth Calculation

总资产 = 当前经费 + 所有持有星球的交公价总和（即按最差情况变现的总价值）。 Net worth = current funds + sum of mortgage values of all owned planets (i.e. worst-case liquidation value).

破产流程

Bankruptcy Process

  1. 经费不足以支付过路费或其他费用时，先尝试出售（交公）自己的星球换取经费。
  2. When unable to pay, first try selling (mortgaging) your planets to raise funds.
  3. 若卖光所有星球仍无法凑齐所需金额，宣布破产，退出游戏。
  4. If selling everything still doesn't cover the debt, declare bankruptcy and exit the game.
  5. 破产者的所有星球归还银行（重置为无人状态）。
  6. All bankrupt player's planets return to the bank (reset to unclaimed).

排名

Rankings

游戏结束后，按总资产从高到低显示所有玩家排名。破产者排在末位。

At game end, all players are ranked by net worth (highest first). Bankrupt players are ranked last.
